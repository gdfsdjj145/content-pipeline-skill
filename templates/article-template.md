# 公众号长文结构公式

> Phase 4 生成的文章结构。适用于公众号/知乎中长内容。

---

## 恐慌 → 反转 → 证据 → 出路

### 第1段：恐慌（前1/3，制造焦虑）

- 开篇前3句：直接给最刺激数据/结论，不铺垫
- 堆数据：至少2-3个信源
- 堆岗位/现象：目标人群的普遍困境
- 堆时间/成本：沉默成本有多大

**示例开头结构**：
```
话题：某个AI趋势/数据
↓ Hook（前3句就要有痛感）
但99%的人不知道的是...
↓ 可信度（个人背书）
过去5个月我用这个方法做了X款产品...
```

### 第2段：反转（中间）

- 「但有一种人不怕...」
- 「这里面藏着机会...」
- 转折点的核心洞察

### 第3段：证据（实战）

- **个人实战数据**（不要放末尾！）
- 真实案例：产品/用户/收入
- 转变故事：之前 vs 之后

### 第4段：出路（方法）

- 可执行的下一步行动
- Input→Build→Ship 或具体步骤清单
- 门槛最低的成功路径

## 内容形式选择

| 选题特性 | 推荐形式 | 理由 |
|---|---|---|
| 观点输出、认知冲突 | 公众号长文 | 短内容装不下 |
| 工具清单、操作教程 | 图文（知乎） | 用户需保存反复查看 |
| 深度分析、多信源 | 公众号长文/知乎回答 | 短内容装不下 |
| 案例复盘、数据展示 | 图文+公众号组合 | 图文放数据，长文讲故事 |
| 热点事件解读 | 公众号 | 快速发布抢时间窗口 |

## 各平台适配

### 公众号版

- 承担信任沉淀和付费转化角色
- 结构：恐慌→反转→证据→出路
- 文末固定引流：{{CTA_TEXT}}

### 知乎版

- 找相关问题回答
- 模板：反差开头 → 正确方法 → 实战数据 → {{ZHIHU_CTA}}

## 标题自检

**基础三类（至少命中一类）**：
- [ ] 赚钱/结果承诺：包含具体数字、收入、产品数量
- [ ] 省时间的工具：承诺清单、模板、可直接用的东西
- [ ] 反差/冲突/反常识：制造认知冲突

**升级检查**：
- [ ] 悬念：标题保留悬念，答案不在标题里
- [ ] 数字冲击：优先用大数字（6000万 > 342）
- [ ] 3秒痛感：开头前3句直接给最刺激数据

**公式：标题卖结果/工具/冲突，内容藏方法论。**
